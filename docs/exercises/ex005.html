Exercício 005

Formulários HTML - Coletando dados do usuário

📚 Tipo

Exercício Básico

⏱️ Duração

20-30 minutos

🎯 Dificuldade

Iniciante

🛠️ Tecnologias

HTML5

🎯 Objetivo

📝 Criando interação com usuários! Neste exercício, você aprenderá a criar formulários HTML para coletar informações e interagir com os visitantes do seu site.

Formulários são a ponte entre seu site e os usuários. Aprenda a criá-los de forma acessível, semântica e com excelente experiência do usuário.

🏗️ Estrutura de Formulários

  • Tag <form> como container
  • Campos de entrada <input>
  • Labels para acessibilidade
  • Organização semântica

🎛️ Tipos de Campos

  • type="text" - Texto simples
  • type="email" - Endereços de email
  • type="password" - Senhas
  • type="submit" - Botões de envio

🎯 Boas Práticas UX

  • Labels descritivos e claros
  • Validação HTML5 nativa
  • Atributos de acessibilidade
  • Feedback visual adequado

💡 Dica de Acessibilidade

Sempre associe labels aos campos usando o atributo for! Isso melhora a acessibilidade e permite que usuários cliquem no label para focar no campo.

💻 Código Completo

📄 index.html 
<!DOCTYPE html>
<html lang="pt-br">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Meu primeiro formulário</title>
</head>
<body>
    <h1>Meu primeiro formulário</h1>
    <form>
        <p>Nome: <input type="text" name="nome" id="nome"></p>
        <p>Sobrenome: <input type="text" name="sobrenome" id="sobrenome"></p>
        <p><input type="submit" value="Enviar"></p>
    </form>
</body>
</html>

🏗️ Estrutura do Formulário

📝 Tag <form>:

A tag <form> é o contêiner principal que agrupa todos os elementos do formulário:

<form action="/processar" method="POST">
    <!-- Elementos do formulário aqui -->
</form>

🔧 Atributos Importantes:

  • action: URL para onde os dados serão enviados
  • method: Método HTTP (GET ou POST)
  • enctype: Tipo de codificação dos dados
  • target: Onde abrir a resposta

📝 Tipos de Campos de Entrada

📝 Text Input

<input type="text">

Campo de texto simples para entrada de dados.

🔒 Password

<input type="password">

Campo para senhas (caracteres ocultos).

📧 Email

<input type="email">

Campo específico para endereços de email.

🔢 Number

<input type="number">

Campo numérico com controles de incremento.

📅 Date

<input type="date">

Seletor de data com calendário.

☑️ Checkbox

<input type="checkbox">

Caixa de seleção para múltiplas opções.

🔘 Radio

<input type="radio">

Botão de opção para escolha única.

🚀 Submit

<input type="submit">

Botão para enviar o formulário.

🏷️ Labels e Acessibilidade

Labels são essenciais para acessibilidade e usabilidade dos formulários:

Usando Labels Corretamente 
<!-- Método 1: Label envolvendo o input -->
<label>
    Nome:
    <input type="text" name="nome">
</label>

<!-- Método 2: Label com atributo 'for' -->
<label for="email">Email:</label>
<input type="email" id="email" name="email">

✅ Benefícios dos Labels:

  • Melhora a acessibilidade para leitores de tela
  • Permite clicar no label para focar o campo
  • Melhora a experiência do usuário
  • Facilita a navegação por teclado

⚙️ Atributos Importantes

🔧 Atributos Comuns:

  • name: Nome do campo (usado no envio)
  • id: Identificador único do elemento
  • value: Valor padrão do campo
  • placeholder: Texto de exemplo/dica
  • required: Campo obrigatório
  • disabled: Campo desabilitado
  • readonly: Campo somente leitura
Exemplo com Atributos 
<input 
    type="email" 
    name="email" 
    id="email"
    placeholder="Digite seu email"
    required
>

🧩 Outros Elementos de Formulário

📝 Textarea:

<textarea name="mensagem" rows="4" cols="50">
Texto padrão aqui...
</textarea>

📋 Select (Lista suspensa):

<select name="cidade">
    <option value="sp">São Paulo</option>
    <option value="rj" selected>Rio de Janeiro</option>
    <option value="mg">Minas Gerais</option>
</select>

🎛️ Fieldset e Legend:

<fieldset>
    <legend>Informações Pessoais</legend>
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome">
</fieldset>

✅ Boas Práticas

🎯 UX/UI:

  • Use labels descritivos e claros
  • Agrupe campos relacionados com fieldset
  • Forneça feedback visual para validação
  • Use placeholders como dicas, não como labels
  • Mantenha formulários simples e focados

♿ Acessibilidade:

  • Sempre associe labels aos campos
  • Use atributos ARIA quando necessário
  • Garanta navegação por teclado
  • Forneça instruções claras
  • Use contraste adequado nas cores

🔒 Segurança:

  • Valide dados no cliente E no servidor
  • Use HTTPS para dados sensíveis
  • Implemente proteção CSRF
  • Sanitize dados de entrada

🚀 Desafios Extras

  1. Crie um formulário de cadastro completo
  2. Adicione validação HTML5 aos campos
  3. Implemente diferentes tipos de input
  4. Use fieldset para agrupar informações
  5. Crie um formulário de contato com textarea
  6. Adicione um select com múltiplas opções
  7. Implemente radio buttons para escolha única
  8. Use checkboxes para múltiplas seleções
